You must gather these unique keys and credentials to connect your project to essential services.

1. ‚òÅÔ∏è MongoDB Atlas Connection String (MONGO_URI)

  1. Create an account and cluster on MongoDB Atlas.
  2. Set up a Database Access User (e.g., my_tripzo_user).
  3. Configure Network Access to allow your IP address.
  4. Copy the connection string and replace the <username> and <password> placeholders.

2. üõ°Ô∏è JWT Secret (JWT_SECRET)

3. üìß Google App Password (EMAIL_PASS) for OTP Service

  1. Go to Google Account Security.
  2. Ensure 2-Step Verification is ON.
  3. Go to App Passwords ‚Üí select Mail and Other (Tripzo App).
  4. Click Generate. The 16-character code is your EMAIL_PASS.

üåê Section G: Global Deployment with Netlify (Frontend Only)

Netlify is the easiest way to deploy your static frontend for worldwide access.

Step 1: Netlify Account Setup

  1. Go to Netlify and click Sign Up.
  2. Select GitHub to connect your account and authorize access to your repositories.

Step 2: Import Project from GitHub

  1. Click "Add new site" → "Import an existing project".
  2. Select **GitHub** and find your **Tripzo** repository from the list.

Step 3: Configure Deployment Settings

SettingValueExplanation
OwnerYour GitHub UsernameDefault value
Branch to deploymainThe production branch
Base directoryLeave emptyThe root of your repository
Build commandLeave emptyNo build process is required for static HTML/CSS/JS
Publish directoryFrontendTripzoCrucial: Tells Netlify which folder contains the public website files.

Step 4: Deploy and Go Live!

Click the ‚ÄúDeploy site‚Äù button.

Netlify will give you a unique URL where your Frontend is live (e.g., https://random-name-12345.netlify.app).

Step 5: Final Steps for a Fully Functional Live App

Since your Netlify site is still looking for the backend on your local machine, you must complete this final step:

  • Deploy your BackendTripzo to a live Node.js hosting service (e.g., Render or Railway) to get a production API URL (e.g., https://tripzo-api-live.com).
  • Update the frontend code: **Search and replace** all instances of http://localhost:3000 with your new live API URL in the frontend files (e.g., in services.html and index.html).
  • Commit and push these final frontend changes to GitHub. Netlify will auto-update, and the full application will be functional globally!
